# Design notes

How `gpusw` generalizes a single hard-coded local-SW CUDA kernel into a configurable,
correct, installable aligner — and why each decision was made.

## Goals

1. **Any alignment-scoring input.** Arbitrary query/reference sequences, arbitrary scoring.
2. **Parameter breadth.** Mode, gap costs, alphabet, full substitution matrices, dtype.
3. **Funnel experimental inputs into discrete forms.** Accept the many shapes real data
   arrives in; normalize to the one discrete representation the kernel consumes.
4. **Keep the constraints that made the original work.** Runtime NVRTC compilation (no
   `nvcc`), score-only, one thread per pair, and *bit-exactness* against a CPU reference.

## The discrete form

The kernel consumes two arrays per sequence set:

- `codes` — every sequence's residues encoded to integer codes `0..K` and concatenated
  into one contiguous `uint8` buffer (`K` is the out-of-alphabet sentinel).
- `offsets` — `int32[n+1]` prefix sums; sequence `i` is `codes[offsets[i]:offsets[i+1]]`.

`gpusw.encode.funnel` is the single dispatcher that turns lists / dicts / `(id, seq)`
iterables / `SeqRecord`-likes / FASTA (file, `.gz`, or text) / raw sequences / pre-encoded
buffers into an `Encoded(codes, offsets, ids, lengths, alphabet)`.

## Always-matrix scoring

The kernel scores a cell as `s = SUB[a*NS + b]`, where `SUB` is a small `NS×NS` table and
`NS = len(alphabet) + 1`. Match/mismatch schemes simply build a diagonal `SUB`. Benefits:

- One code path for DNA, protein, and custom alphabets.
- **Correct unknown handling for free.** The extra sentinel row/col encodes the `unknown`
  policy, so two different (or identical) out-of-alphabet symbols never spuriously match —
  the failure mode of a naive `(a == b)` match/mismatch kernel with a shared sentinel code.
- `SUB` is uploaded once as a plain `const int* __restrict__` device argument (cached per
  `(table, device)`), not baked into `__constant__` — no size baking, identical tables share
  one upload.

`SUB` is *data*; only its side length `NS` affects the generated source. So two schemes that
differ only in their substitution values share a compiled module and differ only in the
uploaded table.

## Modes (compile-time, branch-free)

Rows = reference, cols = query. First gap residue costs `gap_open`, each subsequent costs
`gap_extend` (verified against Biopython).

- **local**: clamp `H ≥ 0`, return the running maximum.
- **global**: boundary rows/cols follow the affine ladder `gap_open + (k-1)*gap_extend`;
  return the corner cell.
- **semiglobal**: `free_end_gaps_ref` zeroes the row-0 boundary and lets the answer come
  from the last row; `free_end_gaps_query` does the same for col-0 / the last column. The
  exact mapping was pinned by matching Biopython's `target_end_gap_score` /
  `query_end_gap_score` across all four flag combinations (200/200 each).

Empty-sequence edges are handled explicitly and identically in the kernel and the oracle.

## int16 vs int32

DP math is done in 32-bit `int` registers; only stored cells (`Hp`, `F`, and the output)
narrow to the chosen type. A static two-sided envelope bounds the reachable score range from
the score extremes (`max_pos`, `min_sub`) and the sequence lengths:

- upper ≈ `min(maxQ, maxR) * max_pos`
- lower ≈ `min(gap_open, gap_extend, min_sub) * max(maxQ, maxR)` (global/semiglobal only)

`int16` is chosen when both bounds fit `±30000` (a `-32000` sentinel stays representable),
else `int32`. Forcing `dtype="int16"` past the bound raises `OverflowRiskError`.

## Compile + cache

Source is generated by `kernel.render_source(module_fields, maxq, dtype)` and compiled with
`cupy.RawModule(code, options=("--std=c++14",))`. `MAXQ` is bucketed up to the next 128 so
near-equal query lengths reuse one module. The module cache key is

```
(mode, gap_open, gap_extend, free_q, free_r, ns, maxq_bucket, dtype, compute_capability)
```

Including the GPU architecture keeps a cached module from being reused after a device switch.

## Correctness strategy

`reference.py` is a pure-NumPy affine-Gotoh oracle that mirrors the kernel exactly (same
recurrence, gap convention, `SUB` indexing, free-end-gap handling, empty-edge rules). It is
anchored to Biopython at test time and is the assertion target for the GPU tests, so the GPU
path is validated transitively against Biopython without a GPU in CI.

## Deliberate non-goals (v0.1.0)

- **No query/reference auto-swap.** The query is always the inner DP buffer; if your shorter
  set is the references, pass it as `queries`. Auto-swapping interacts subtly with semiglobal
  asymmetry, so it is left out rather than risk a wrong free-edge.
- **Score-only.** Percent identity / coordinates are recovered for surviving hits with a
  cheap CPU re-align.
- **One thread per pair.** Intra-sequence (warp-parallel) DP and SIMD int16×2 packing are
  future throughput work.
